# Security policy

## Supported versions

The latest `0.4.x` release receives security fixes. Upgrade to the newest PyPI release before
reporting an issue that may already be resolved.

## Report a vulnerability

Use GitHub's
[private vulnerability reporting](https://github.com/rukouai/aimlib-python/security/advisories/new).
Do not open a public issue for a suspected vulnerability.

Include the installed SDK version, Python version, a minimal reproduction using mock data, and the
security impact. Never include an aimlib API key, credential-bearing proxy URL, browser connection
token, cookie, customer data, full browsing history, or production device details.

We will acknowledge the report, investigate it, and coordinate remediation and disclosure with the
reporter. Please do not test against accounts, devices, or data you do not own or have explicit
permission to assess.
